# knowledge-mining

自动扫描飞书知识库，为文档打标签并聚合方法论，输出可直接作为 skill 原型的场景知识文档。面向游戏运营和数据分析的 Know-how 沉淀场景。

## When to Use This Skill
当用户需要：
- 批量整理飞书知识库中的文档
- 为大量文档自动打标签分类
- 从知识库中提炼可复用的分析方法论
- 生成结构化的 skill 知识文档
- 沉淀游戏运营或数据分析的 Know-how

## 前置依赖 (Prerequisites)
- 飞书 MCP 工具：需配置飞书文档读写权限
- 用户提供的根文档链接：作为存储所有产出的父节点
- 文档访问权限：用户需对目标知识库有读取权限

## 执行步骤 (Instructions)
### 准备阶段
要求用户提供一个飞书文档链接（作为存储根节点），后续所有产出都以子文档形式挂在该文档下。
### 第一阶段：全量扫描
- 递归遍历用户提供的 wiki 链接，识别目录型文档并展开子文档。
- 生成完整文档清单（标题、链接、预估类型）。
### 第二阶段：基于标题初筛 → 写入飞书
按标题关键词分两类处理：
标题已明确（含"流失/付费/留存/转化"等词）→ 直接打标签
需全文识别（泛化标题如"XX解决方案/XX合集"）→ 标记待深读
初筛结果写入子文档《文档标签初筛结果》。
### 第三阶段：确认全文读取范围
让用户选择哪些"需全文识别"的文档要继续深读（支持指定编号、全部读取、按建议读取）。
### 第四阶段：全文精细打标 → 写入飞书
对确认的文档全文读取（自动分页），按三个维度打标：
维度	示例值
行业/品类	MMO、卡牌、SLG、通用...
业务场景	流失召回、付费转化、留存提升...
问题类型	诊断分析、策略制定、效果评估...
结果写入子文档《文档标签》。
### 第五阶段：场景聚合与典型问题提炼 → 写入飞书
按业务场景分组，提炼每个场景的 3-5 个核心典型问题。
写入子文档《场景聚合与典型问题提炼》。
### 第六阶段：确认深挖场景
让用户选择要为哪些场景生成完整的 skill 知识文档。
### 第七阶段：生成场景 Skill 知识文档
为每个选定场景单独建一篇子文档，内容高度结构化：
- 场景概述
- 典型问题（业务角度表述）
- 分析框架（每步：目标 + 具体做法 + 工具 + 注意事项）
- 关键指标体系（含计算方式和经验阈值）
- Know-how + 常见坑点（含来源文档）
- 工具/方法对照表
- 参考案例（脱敏后可复用）

## 输出标准 (Expected Output)
### 产出文档结构
根文档（用户提供）
├── 文档标签初筛结果
├── 文档标签
├── 场景聚合与典型问题提炼
└── [场景名称] Skill 知识文档（可多个）

### Skill 知识文档质量标准
- 输出目标是 skill 原型，而不是文档摘要
- 以"资深分析专家"视角提炼通用解题框架，而非复述个案
- 产出内容要能让任何分析师读完之后知道"下一步做什么、看什么数据、用什么工具"
